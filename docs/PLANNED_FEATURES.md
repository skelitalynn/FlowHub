# FlowHub 规划功能文档

本文件用于记录尚未实现、但计划逐步落地的功能点，便于后续迭代与分阶段交付。

## 1. AI / Agent
### 1.1 语义检索（Embedding + 向量检索）
- 目标：支持“自然语言搜索店铺/内容”
- 依赖：Embedding 模型 + 向量库（Milvus/Qdrant/pgvector）
- 输出：按语义相关性排序的店铺列表

### 1.2 推荐理由生成（LLM）
- 目标：为推荐结果生成可解释理由
- 依赖：LLM + 推荐特征输入（评分/评论/距离/热度）
- 输出：每条推荐带可读理由

### 1.3 本地生活助手 Agent
- 目标：多步骤任务（查店铺 → 比较 → 下单 → 查询进度）
- 依赖：Function Calling / Tool 调用层
- 输出：对话式完成任务

## 2. 实时趋势与统计
### 2.1 热度榜
- 目标：实时更新热门店铺/优惠券榜单
- 依赖：Redis Streams / Kafka / Flink
- 输出：实时榜单接口与页面

### 2.2 趋势监控
- 目标：统计访问/下单/收藏趋势
- 依赖：流式统计 + 时序存储
- 输出：趋势曲线与告警

## 3. 工程化
### 3.1 Docker Compose 一键启动
- 目标：MySQL + Redis + 应用一键拉起
- 输出：`docker-compose.yaml`

### 3.2 可观测性
- 目标：日志结构化 + TraceId + 关键指标
- 依赖：Logback JSON / Micrometer / Prometheus
- 输出：监控面板与统一日志

### 3.3 测试与压测
- 目标：关键链路自动化测试与压测
- 依赖：JUnit / RestAssured / JMeter
- 输出：测试报告与性能基线

## 4. 功能增强
### 4.1 图片上传与对象存储
- 目标：将图片上传从本地路径迁移到对象存储
- 依赖：MinIO / OSS / COS

### 4.2 权限与角色
- 目标：用户角色与权限控制
- 依赖：Spring Security / RBAC

### 4.3 推荐系统基础版本
- 目标：基于行为的简单推荐（浏览/点赞/收藏）
- 依赖：行为采集与特征计算
